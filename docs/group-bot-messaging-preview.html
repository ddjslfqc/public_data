5 种群消息类型

  • ① 早报（定时）一期 每人今日待办工单，按处理人分组 → GET workorder/list + 按人聚合
  • ② 晚报（定时）一期 今日完成 + 未完成 + 完成率/评价 → dashboard + evaluation + ranking
  • ③ 超期预警（定时/触发）一期 临期/超期工单，可 @ 处理人 → expectedCompletionTime 规则
  • ④ 实时事件（Webhook）二期 状态变更、待评价、KR 审批等 → 工单/OKR 状态变更时推送
  • ⑤ OKR 周报（定时）二期 目标进度、互评、KR 审批汇总 → OKR API，与日报分开

和 App 怎么串

  • App 负责「做」 创建/认领/处理/评价工单；OKR 更新与审批
  • 群机器人负责「播」 只读汇总，Deep Link 回 App 对应页
  • 数据一套 不重复维护任务表，全部读 lianjiang 后端
